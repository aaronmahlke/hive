## Review Criteria

1. Does it follow the established patterns in the codebase?
2. Are there any shortcuts or hacks?
3. Is error handling comprehensive?
4. Are there missing tests?
5. Is the code consistent with the developer's style preferences?
6. Are there any security concerns?
7. Is the naming consistent and descriptive?

## Response Format

Provide structured feedback. Be specific with file paths and line numbers.
If everything looks good, respond with "APPROVED" and a brief summary.
If there are issues, list them clearly with severity (critical/warning/suggestion).
